## Детальный анализ: Достойные для парсинга сайты

### 1. Semantic Scholar ✅ РЕКОМЕНДОВАН

**URL:** https://www.semanticscholar.org
**API:** `https://api.semanticscholar.org/graph/v1/paper/search`
**Авторизация:** Не требуется (rate limit: ~1000 req/s shared)

**Возможности:**
- 200M+ статей из всех областей
- TL;DR саммари для многих статей
- Граф цитирований и ссылок
- Фильтры: год, область, venue
- Bulk API для массовой загрузки

**Применение для OpenClaw:**
- Регулярный мониторинг новых статей по RL, LoRA, multi-agent
- Отслеживание цитирований ключевых работ
- Автоматическое обновление knowledge base

### 2. Papers With Code ✅ РЕКОМЕНДОВАН

**URL:** https://paperswithcode.com
**API:** `https://paperswithcode.com/api/v1/papers/`
**Авторизация:** Не требуется

**Возможности:**
- Trending статьи с кодом на GitHub
- Бенчмарки и датасеты
- Связь статья → репозиторий → результаты
- Фильтры по задачам (task), методам, датасетам

**Применение для OpenClaw:**
- Найти код реализации для каждой техники
- Бенчмаркинг наших моделей vs SOTA
- Автоматический мониторинг новых GRPO/LoRA реализаций

### 3. arXiv ✅ РЕКОМЕНДОВАН

**URL:** https://arxiv.org
**API:** `https://export.arxiv.org/api/query` (Atom XML)
**Авторизация:** Не требуется

**Возможности:**
- Все AI/ML препринты в реальном времени
- Полнотекстовые аннотации
- Категории: cs.AI, cs.LG, cs.CL, cs.MA
- Сортировка по дате, релевантности

**Применение для OpenClaw:**
- Ежедневный мониторинг новых статей cs.AI + cs.LG
- Автоматический фильтр по ключевым словам
- Интеграция с pipeline_executor для автоматического анализа

### 4. HuggingFace Papers ✅ РЕКОМЕНДОВАН

**URL:** https://huggingface.co/papers
**API:** `https://huggingface.co/api/daily_papers`
**Авторизация:** Не требуется

**Возможности:**
- Trending статьи среди ML-сообщества
- Рейтинг по "likes" и обсуждениям
- Связь с моделями и датасетами на HF
- Daily / weekly / monthly подборки

**Применение для OpenClaw:**
- Отслеживание самых обсуждаемых работ
- Быстрый доступ к связанным моделям и датасетам
- Индикатор того, что стоит внедрять

## Сайты для ручного использования (без парсера)

### AlphaXiv — 🟡 Только ручной анализ
Отличная платформа для чтения и обсуждения arXiv статей с комментариями.
Нет API → используем arXiv API для получения самих статей.

### Emergent Mind — 🟡 Ценный, но без API
AI-саммари и видео-обзоры для cs.AI статей. Используем вручную для
быстрого понимания сложных работ. Синтез из нескольких бумаг.

### Connected Papers — 🟡 Визуальный анализ
Графы связанных работ. Незаменим для ручного исследования кластеров
статей. Используем для планирования направлений исследования.

## Реализованный парсер

**Файл:** `scripts/research_paper_parser.py`

```bash
# Запуск парсера (требует доступ к интернету)
python scripts/research_paper_parser.py --limit 20

# С пользовательскими темами
python scripts/research_paper_parser.py --topics "GRPO,LoRA,multi-agent" --limit 10

# Только конкретный сайт (через фильтр тем)
python scripts/research_paper_parser.py --topics "reinforcement learning" --limit 20
```

**Поддерживаемые сайты:** Semantic Scholar, Papers With Code, arXiv, HuggingFace Papers
**Формат выхода:** Markdown + JSON в `docs/ru/research/{site_name}/`
**Частота запуска:** Рекомендуется еженедельно (cron)

## Рекомендация: Автоматический мониторинг

Для постоянного обновления knowledge base рекомендуется:

1. **Еженедельный cron** для `research_paper_parser.py`
2. **Интеграция с Telegram** — уведомления о новых релевантных статьях
3. **Pipeline Executor** — автоматический анализ новых статей через Planner
4. **Memory GC** — сохранение ключевых инсайтов в Cold_Memory

```bash
# Пример cron (каждое воскресенье в 03:00)
0 3 * * 0 cd /path/to/openclaw_bot && python scripts/research_paper_parser.py --limit 20
```

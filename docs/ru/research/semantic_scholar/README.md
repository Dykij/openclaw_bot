## Подробные описания

### 1. DeepSeek-R1: Incentivizing Reasoning in LLMs via RL

- **Авторы:** DeepSeek-AI et al.
- **Источник:** Semantic Scholar
- **Дата:** 2025
- **Ссылка:** [arXiv:2501.12948](https://arxiv.org/abs/2501.12948)
- **Релевантность для OpenClaw:** 9/10

**Аннотация:**
Работа демонстрирует, что чисто RL-подход (без аннотаций человека) способен обучить LLM продвинутым рассуждениям. DeepSeek-R1 достигает результатов, сопоставимых с OpenAI o1, используя только вознаграждения на основе правильности ответа. Модель обучается цепочечному рассуждению (chain-of-thought) напрямую через reinforcement learning.

**Применение к OpenClaw Bot:**
- Внедрение RL-обучения без ручных аннотаций для улучшения рассуждений бота
- Использование reward-сигналов на основе правильности для автоматического обучения на 16GB VRAM
- Адаптация подхода «cold-start → RL fine-tuning» для OpenClaw-агента

### 2. OpenClaw-RL: Train Any Agent Simply by Talking

- **Авторы:** OpenClaw Team et al.
- **Источник:** Semantic Scholar
- **Дата:** 2026
- **Ссылка:** [arXiv:2603.10165](https://arxiv.org/abs/2603.10165)
- **Релевантность для OpenClaw:** 10/10

**Аннотация:**
Прямая RL-методология для обучения персональных агентов через диалог. Пользователь описывает желаемое поведение на естественном языке, а агент обучается онлайн через reinforcement learning без предварительной разметки данных. Система масштабируется от простых задач до сложных многоэтапных рабочих процессов.

**Применение к OpenClaw Bot:**
- Основополагающая работа для архитектуры OpenClaw — прямая интеграция online RL
- Обучение бота новым навыкам через разговор без программирования
- Реализация цикла «диалог → reward → обновление политики» в production

### 3. QLoRA: Efficient Finetuning of Quantized LLMs

- **Авторы:** Dettmers et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2305.14314](https://arxiv.org/abs/2305.14314)
- **Релевантность для OpenClaw:** 9/10

**Аннотация:**
QLoRA позволяет файнтюнить 65B-модель на одной GPU с 48GB памяти через 4-битную квантизацию и Low-Rank адаптеры. Метод вводит NormalFloat4 — новый тип данных, оптимальный для нормально распределённых весов, и двойную квантизацию для экономии памяти. Результаты приближаются к полному fine-tuning при малой доле потребления ресурсов.

**Применение к OpenClaw Bot:**
- Файнтюнинг моделей на RTX 5060 Ti (16GB VRAM) — ключевое ограничение OpenClaw
- Применение 4-битной квантизации для обучения моделей до 13B параметров локально
- Оптимизация памяти через двойную квантизацию при последовательной загрузке моделей

### 4. LoRA: Low-Rank Adaptation of Large Language Models

- **Авторы:** Hu et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2106.09685](https://arxiv.org/abs/2106.09685)
- **Релевантность для OpenClaw:** 9/10

**Аннотация:**
Основополагающая работа по Low-Rank Adaptation — замораживание предобученных весов и обучение только низкоранговых матриц-декомпозиций. LoRA снижает количество обучаемых параметров в 10 000 раз и требования к GPU-памяти в 3 раза без потери качества. Адаптеры легко переключаются между задачами.

**Применение к OpenClaw Bot:**
- Базовая технология для всех адаптаций моделей в OpenClaw (навыки, каналы, персонализация)
- Быстрое переключение LoRA-адаптеров для разных пользовательских задач
- Хранение компактных адаптеров вместо полных копий модели

### 5. Self-Play Fine-Tuning Converts Weak LMs to Strong LMs

- **Авторы:** Chen et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2401.01335](https://arxiv.org/abs/2401.01335)
- **Релевантность для OpenClaw:** 8/10

**Аннотация:**
SPIN (Self-Play Fine-Tuning) использует самоигру для улучшения слабых LLM без дополнительных данных или сильных моделей-учителей. Модель играет против своей предыдущей версии, что создаёт прогрессивный цикл улучшения. Метод показывает значительное улучшение на бенчмарках после нескольких итераций.

**Применение к OpenClaw Bot:**
- Итеративное самоулучшение бота без внешних API и разметки
- Автономный цикл улучшения «генерация → оценка → обучение» на локальном железе
- Масштабируемое улучшение качества ответов без увеличения размера модели

### 6. Direct Preference Optimization: Your LM is Secretly a Reward Model

- **Авторы:** Rafailov et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2305.18290](https://arxiv.org/abs/2305.18290)
- **Релевантность для OpenClaw:** 8/10

**Аннотация:**
DPO упрощает выравнивание LLM, заменяя отдельную reward-модель на прямую оптимизацию предпочтений. Метод переформулирует задачу RLHF как простую задачу классификации на парах предпочтений. DPO стабильнее и проще в реализации, чем PPO, при сопоставимом качестве.

**Применение к OpenClaw Bot:**
- Упрощение пайплайна выравнивания — не нужна отдельная reward-модель
- Обучение на пользовательских предпочтениях (лайк/дизлайк ответов)
- Экономия VRAM: один пайплайн вместо двух моделей

### 7. Toolformer: Language Models Can Teach Themselves to Use Tools

- **Авторы:** Schick et al.
- **Источник:** Semantic Scholar
- **Дата:** 2023
- **Ссылка:** [arXiv:2302.04761](https://arxiv.org/abs/2302.04761)
- **Релевантность для OpenClaw:** 8/10

**Аннотация:**
Toolformer обучает языковые модели самостоятельно решать, когда и какие API-инструменты вызывать. Модель учится вставлять вызовы API в текст, используя самосупервизию. Подход не требует ручной аннотации вызовов инструментов и работает с произвольным набором API.

**Применение к OpenClaw Bot:**
- Автоматическое обучение бота использовать новые MCP-инструменты
- Самостоятельное определение, когда вызывать внешние API
- Расширение инструментального набора без ручной настройки каждого инструмента

### 8. Constitutional AI: Harmlessness from AI Feedback

- **Авторы:** Bai et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2212.08073](https://arxiv.org/abs/2212.08073)
- **Релевантность для OpenClaw:** 7/10

**Аннотация:**
Constitutional AI заменяет человеческую обратную связь набором принципов (конституцией), по которой модель оценивает и улучшает свои ответы. Метод сочетает самокритику (self-critique) и RL на основе AI-фидбэка (RLAIF). Подход масштабируется без постоянного привлечения аннотаторов.

**Применение к OpenClaw Bot:**
- Определение «конституции» бота — набор принципов поведения
- Автоматическая фильтрация вредных ответов без ручной модерации
- RLAIF для автономного выравнивания на пользовательском железе

### 9. ReAct: Synergizing Reasoning and Acting in Language Models

- **Авторы:** Yao et al.
- **Источник:** Semantic Scholar
- **Дата:** 2023
- **Ссылка:** [arXiv:2210.03629](https://arxiv.org/abs/2210.03629)
- **Релевантность для OpenClaw:** 8/10

**Аннотация:**
ReAct объединяет рассуждение (reasoning) и действие (acting) в единый цикл. Модель чередует генерацию мыслей и выполнение действий, что значительно улучшает результаты на задачах, требующих взаимодействия с внешней средой. Паттерн Thought → Action → Observation стал стандартом для агентных систем.

**Применение к OpenClaw Bot:**
- Основа агентной архитектуры OpenClaw: цикл «Мысль → Действие → Наблюдение»
- Интеграция с MCP-инструментами через ReAct-паттерн
- Улучшение качества многоэтапных задач (поиск, анализ, генерация)

### 10. Reflexion: Language Agents with Verbal Reinforcement Learning

- **Авторы:** Shinn et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2303.11366](https://arxiv.org/abs/2303.11366)
- **Релевантность для OpenClaw:** 8/10

**Аннотация:**
Reflexion использует вербальную рефлексию вместо числовых reward-сигналов для обучения агентов. Агент анализирует свои ошибки в текстовой форме и сохраняет выводы в памяти для будущих попыток. Метод значительно улучшает результаты без обновления весов модели.

**Применение к OpenClaw Bot:**
- Механизм самоанализа ошибок и обучения на собственном опыте
- Текстовая память о прошлых ошибках — совместимо с MemGPT-подходом
- Улучшение без fine-tuning: идеально для ограниченного VRAM

### 16. Mixture of Agents Enhances LLM Capabilities

- **Авторы:** Wang et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2406.04692](https://arxiv.org/abs/2406.04692)
- **Релевантность для OpenClaw:** 8/10

**Аннотация:**
Mixture of Agents (MoA) использует коллаборативность нескольких LLM, где каждый слой агентов улучшает ответы предыдущего слоя. Архитектура достигает 65.8% на AlpacaEval 2.0, превосходя GPT-4o. MoA демонстрирует, что комбинация слабых моделей может превзойти одну сильную.

**Применение к OpenClaw Bot:**
- Многослойная агрегация ответов от нескольких локальных моделей
- Улучшение качества без увеличения размера одной модели
- Последовательная загрузка моделей (ограничение 16GB VRAM) с MoA-агрегацией

### 18. Scaling Data-Constrained Language Models

- **Авторы:** Muennighoff et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2305.16264](https://arxiv.org/abs/2305.16264)
- **Релевантность для OpenClaw:** 7/10

**Аннотация:**
Исследование масштабирования LLM при ограниченных данных: повторение данных до 4 эпох почти не снижает качество. Работа устанавливает новые скейлинг-законы для режима дефицита данных и показывает, что фильтрация по качеству эффективнее, чем увеличение объёма.

**Применение к OpenClaw Bot:**
- Стратегия обучения при ограниченных доменных данных для специализации бота
- Оптимальные параметры повторения данных при fine-tuning
- Качество данных важнее объёма — принцип курации данных для OpenClaw

### 20. Phi-3 Technical Report: A Highly Capable Language Model Locally

- **Авторы:** Microsoft et al.
- **Источник:** Semantic Scholar
- **Дата:** 2024
- **Ссылка:** [arXiv:2404.14219](https://arxiv.org/abs/2404.14219)
- **Релевантность для OpenClaw:** 8/10

**Аннотация:**
Phi-3 — семейство малых моделей (3.8B–14B), обученных на высококачественных данных. Phi-3-mini (3.8B) достигает уровня Mixtral 8x7B и GPT-3.5. Ключ к успеху — курация обучающих данных через фильтрацию и синтетические данные.

**Применение к OpenClaw Bot:**
- Phi-3-mini идеально подходит для 16GB VRAM — компактная и мощная
- Стратегия курации данных для обучения малых моделей OpenClaw
- Доказательство, что малые модели с хорошими данными сравнимы с большими

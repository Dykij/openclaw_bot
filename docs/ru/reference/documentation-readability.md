# Улучшение читаемости документации OpenClaw

Это руководство описывает конкретные методы улучшения читаемости документации OpenClaw — как для людей, так и для AI моделей (локальных и облачных). Все рекомендации основаны на анализе 652+ документов проекта.

## Текущее состояние документации

### Что уже хорошо реализовано

| Аспект | Статус | Описание |
| ------ | ------ | -------- |
| YAML frontmatter | ✅ | Все английские документы имеют `summary`, `read_when`, `title` |
| llms.txt | ✅ | Карта документации для AI агентов на `docs.openclaw.ai/llms.txt` |
| llms-full.txt | ✅ | Расширенное описание для Deep Research |
| Hubs-страница | ✅ | `/start/hubs` — полный каталог всех документов |
| RAG-пайплайн | ✅ | `scripts/doc_ingester.py` для векторного поиска |
| memory_search | ✅ | Семантический поиск по документам workspace |
| Mintlify хостинг | ✅ | `docs.openclaw.ai` с навигацией и поиском |
| i18n пайплайн | ✅ | zh-CN (китайский), ja (японский) переводы |

### Что можно улучшить

| Аспект | Проблема | Решение |
| ------ | -------- | ------- |
| Русская документация | Отсутствует | Создать `docs/ru/` с ключевыми гайдами |

| Примеры для 8GB VRAM | Мало | Добавить конфигурации для ограниченного GPU |

## Принципы AI-оптимизированной документации

### 1. Структура "Перевёрнутая пирамида"

AI модели (особенно RAG-системы) часто читают только начало документа или секции. Размещайте самую важную информацию первой:

```markdown
<!-- ❌ Плохо: ключевая информация в конце -->
## Установка Ollama

Ollama — это инструмент для локального запуска LLM моделей.
Он был создан в 2023 году и поддерживает множество форматов.
Работает на Linux, macOS и Windows через WSL2.

Для установки выполните:
curl -fsSL https://ollama.ai/install.sh | sh

<!-- ✅ Хорошо: команда первой, контекст потом -->
## Установка Ollama

Установите Ollama одной командой:
curl -fsSL https://ollama.ai/install.sh | sh

Ollama запускает LLM модели локально на Linux, macOS и Windows (WSL2).
```

### 2. Самодостаточные секции

Каждый H2-раздел должен быть понятен отдельно от остальной страницы. RAG-система может извлечь только эту секцию:

```markdown
<!-- ❌ Плохо: зависит от предыдущей секции -->
## Конфигурация
Настройте его через файл конфигурации...

<!-- ✅ Хорошо: контекст в каждой секции -->
## Конфигурация Ollama в OpenClaw
Настройте Ollama как провайдера в `~/.openclaw/openclaw.json`:
```

### 3. Описательные заголовки

Заголовки — основа навигации для AI и людей:

```markdown
<!-- ❌ Плохо -->
## Настройка
## Использование
## Проблемы

<!-- ✅ Хорошо -->
## Как настроить Ollama для OpenClaw
## Как отправить сообщение через CLI
## Решение проблем с подключением WhatsApp
```

### 4. Явные определения сущностей

При первом упоминании понятия — определите его:

```markdown
<!-- ❌ Плохо: предполагает знание -->
Gateway слушает на порту 18789.

<!-- ✅ Хорошо: определяет понятие -->
**Gateway** — это центральный процесс-демон OpenClaw, который слушает
WebSocket-подключения на порту 18789 и маршрутизирует сообщения между
каналами (WhatsApp, Telegram и др.) и AI агентами.
```

### 5. Таблицы вместо списков для сравнений

Таблицы парсятся AI моделями надёжнее:

```markdown
<!-- ❌ Плохо: сложно сравнивать -->
- Brave: быстрый, бесплатный тариф, структурированные результаты
- Perplexity: AI-синтез, цитаты, нужен API-ключ
- Gemini: Google Search, AI-синтез, нужен API-ключ

<!-- ✅ Хорошо: легко сравнить -->
| Провайдер | Скорость | Цена | Тип результатов |
| --------- | -------- | ---- | --------------- |
| Brave | Быстрый | Бесплатный тариф | Структурированные |
| Perplexity | Средний | Платный | AI-синтез с цитатами |
| Gemini | Средний | Платный | AI-синтез через Google |
```

### 6. FAQ-секции для сложных тем

FAQ-формат идеально подходит для AI-поиска — модели часто формулируют запросы как вопросы:

```markdown
## FAQ

### Как переключить модель в OpenClaw?

Используйте CLI:
openclaw config set agents.defaults.model.primary "ollama/llama3.3:8b"

Или измените `~/.openclaw/openclaw.json`:
{
  agents: {
    defaults: {
      model: { primary: "ollama/llama3.3:8b" }
    }
  }
}

### Почему Ollama не отвечает?

Проверьте статус: `systemctl status ollama` или `ollama list`.
Частые причины: сервис не запущен, порт 11434 занят, нехватка VRAM.
```

### 7. Код с тегами языка и комментариями

Плохой пример (нет тега языка, нет комментариев):

````
openclaw config set tools.web.search.enabled true
````

Хороший пример (тег + комментарий):

```bash
# Включить web_search для Deep Research через интернет
openclaw config set tools.web.search.enabled true
```

## Чеклист для новой документации

Перед публикацией нового документа проверьте:

### Метаданные

- [ ] YAML frontmatter с `summary`, `read_when`, `title`
- [ ] `summary` — одно точное предложение, описывающее содержание
- [ ] `read_when` — 2-3 конкретных сценария (не абстрактных)
- [ ] Страница добавлена в [Docs hubs](/start/hubs)
- [ ] Страница указана в `llms.txt`

### Структура

- [ ] Заголовки описательные (H1 > H2 > H3 иерархия)
- [ ] Каждая H2-секция самодостаточна
- [ ] Первый абзац содержит ключевую информацию
- [ ] Перекрёстные ссылки на связанные документы

### Контент

- [ ] Код с тегами языка (` ```bash `, ` ```json5 `, ` ```typescript `)
- [ ] Ключевые понятия определены при первом упоминании
- [ ] Нет неоднозначных местоимений ("он", "это" → конкретное название)
- [ ] Рабочие примеры (не абстрактные описания)
- [ ] FAQ-секция для сложных тем
- [ ] Таблицы для сравнений

### Для локального поиска

- [ ] Документ доступен в `memory/` workspace агента
- [ ] `openclaw doctor` подтверждает доступность memory_search
- [ ] `memory_search` находит документ по ключевым терминам

## Оптимизация для memory_search

### Как memory_search индексирует документы

```
Markdown файл
  → Разбивка на чанки (400 токенов, 80 перекрытие)
  → Эмбеддинг каждого чанка (векторное представление)
  → Сохранение в SQLite + sqlite-vec
  → При запросе: cosine similarity → Top-K результаты
```

### Рекомендации по структуре документов для поиска

1. **Размер параграфов**: каждый параграф — одна законченная мысль (150-400 токенов)
2. **Заголовки-якоря**: каждый H2/H3 заголовок включает ключевые термины
3. **Первое предложение**: суть раздела в первом предложении
4. **Повторение ключевых слов**: используйте основные термины 2-3 раза в секции
5. **Избегайте длинных списков**: более 10 пунктов разбивайте на подсекции

### Пример оптимального документа

````markdown
---
summary: "Настройка Ollama как локального провайдера моделей для OpenClaw"
read_when:
  - Вы устанавливаете Ollama для OpenClaw
  - Вы хотите использовать локальные модели без API-ключей
title: "Ollama"
---

# Ollama — локальный провайдер моделей

Ollama позволяет запускать LLM модели локально на вашем компьютере.
OpenClaw автоматически обнаруживает модели Ollama и предоставляет
их через единый API без внешних зависимостей.

## Быстрый старт с Ollama

Установите Ollama и загрузите модель:

```bash
curl -fsSL https://ollama.ai/install.sh | sh
ollama pull qwen2.5-coder:7b
```

Настройте OpenClaw:

```bash
openclaw config set models.providers.ollama.apiKey "ollama-local"
```

## FAQ

### Какие модели поддерживает Ollama в OpenClaw?

OpenClaw поддерживает все модели с поддержкой `tools` capability.
Проверьте доступные модели: `ollama list && openclaw models list`.

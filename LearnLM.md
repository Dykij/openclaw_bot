# 🧠 LearnLM — Роадмап обучения моделей OpenClaw

> **Цель:** дообучить модели бота на своих данных через QLoRA, чтобы они лучше отвечали  
> на задачи DMarket, CS2-торговли и управления ботом.  
> **Железо:** RTX 5060 Ti 16GB · WSL2 Ubuntu · vLLM  
> **Модели:** Qwen2.5-Coder-14B-AWQ (главная) · Qwen2.5-Coder-7B-AWQ (тренировочная)

---

## 📅 ФАЗА 0 — Установка зависимостей (один раз, ~20 минут)

Это фундамент всего пайплайна обучения. Без правильно настроенного окружения ни один из следующих шагов не заработает. Выполни **один раз** — потом забудь.

### Шаг 0.1 — Проверь что WSL2 и CUDA доступны

```bash
# В PowerShell — убедись что WSL видит GPU:
wsl nvidia-smi
```

Ожидаемый вывод (пример):

```
+-----------------------------------------------------------------------------+
| NVIDIA-SMI 560.xx    Driver Version: 560.xx    CUDA Version: 12.4          |
+-----------------------------------------------------------------------------+
| GPU 0: NVIDIA GeForce RTX 5060 Ti    16376MiB                              |
+-----------------------------------------------------------------------------+
```

Если `nvidia-smi` не найдена или GPU не видна — сначала установи [CUDA Toolkit для WSL2](https://developer.nvidia.com/cuda-downloads) и перезапусти WSL командой `wsl --shutdown`.

### Шаг 0.2 — Проверь Python и виртуальное окружение

```bash
# Убедись что vllm_env существует и Python актуален:
wsl bash -c "source /mnt/d/vllm_env/bin/activate && python --version && pip --version"
```

Если окружения нет — создай:

```bash
wsl bash -c "python3.11 -m venv /mnt/d/vllm_env && source /mnt/d/vllm_env/bin/activate && pip install --upgrade pip"
```

### Шаг 0.3 — Установи зависимости для обучения

```bash
# Открой WSL и выполни:
wsl bash /mnt/d/openclaw_bot/openclaw_bot/scripts/install_training_deps.sh
```

Что делает скрипт внутри (для понимания):

```bash
pip install unsloth[cu124]        # Ускоренный QLoRA (в 2x быстрее обычного)
pip install trl>=0.11             # Trainer для LoRA — SFTTrainer
pip install peft>=0.13            # LoRA адаптеры Hugging Face
pip install datasets>=2.19        # Загрузка и обработка датасетов
pip install rouge_score           # Метрика качества ROUGE-1
pip install bitsandbytes>=0.43    # 4-bit квантизация
pip install accelerate            # Ускорение обучения через FSDP/DeepSpeed
```

> ⚠️ **Unsloth** — ключевая библиотека. Она оптимизирует QLoRA под RTX для экономии VRAM.  
> Без неё 14B модель не влезет в 16GB. Убедись что версия совместима с CUDA 12.4.

### Шаг 0.4 — Проверка что всё установилось

```bash
wsl bash -c "source /mnt/d/vllm_env/bin/activate && python -c 'import unsloth, trl, peft; print(\"OK — готово к обучению\")'"
```

Дополнительная проверка — тест GPU через PyTorch:

```bash
wsl bash -c "source /mnt/d/vllm_env/bin/activate && python -c \
  'import torch; print(\"CUDA:\", torch.cuda.is_available()); \
   print(\"GPU:\", torch.cuda.get_device_name(0)); \
   print(\"VRAM:\", round(torch.cuda.get_device_properties(0).total_memory/1e9,1), \"GB\")'"
```

Ожидаемый вывод:

```
CUDA: True
GPU: NVIDIA GeForce RTX 5060 Ti
VRAM: 16.3 GB
```

## 📅 ФАЗА 1 — Сбор данных (Недели 1–3)

Качество датасета определяет 80% результата обучения. Чем разнообразнее и правдивее примеры — тем умнее получится модель.

### Каждый день — 0 минут (автоматически)

Просто **пользуйся ботом как обычно.** Каждый диалог с ботом записывается в лог и станет примером для обучения. Не нужно ничего специально делать — логи пишутся автоматически в `logs/bot_current.log`.

**Чем больше используешь бота → тем лучше обучится модель.**

---

### Каждые 2–3 дня — 5 минут

**Шаг 1.1.** Собери данные из логов:

```powershell
# В Windows PowerShell (корень проекта):
python scripts/collect_training_data.py
```

Посмотри на вывод:

```
Raw pairs found: 47
After quality filter: 31
Total samples : 31
Avg prompt len: 142 chars
Avg output len: 840 chars
```

**Что означает "quality filter"?** Скрипт автоматически отфильтровывает:

- Диалоги короче 50 символов (слишком мало информации)
- Ответы типа "ошибка", "недоступно", "не знаю" (бесполезные для обучения)
- Дублирующиеся запросы (оставляет только лучший ответ)
- Технические служебные сообщения бота (не несут смысловой нагрузки)

---

**Шаг 1.2.** Добавь 2–5 примеров вручную (самое важное!).

Открой файл `data/training/raw_dialogues.jsonl` и допиши хорошие диалоги.  
Каждый пример — одна строка в формате JSON с тремя полями:

- `instruction` — что просит пользователь (чёткая задача)
- `input` — дополнительный контекст (может быть пустым `""`)
- `output` — идеальный ответ бота (развёрнутый, полезный, с числами)

Evaluating adapter: openclaw-7b-v1

ROUGE-2: 0.28

Интерпретация результата:

|---------|-----------|---------|

### Шаг 4.2 — Запуск обучения на ночь (5 минут настройки + 8–10 часов)

Запускай **на ночь** — комп будет занят до утра:

```bash
wsl bash -c "source /mnt/d/vllm_env/bin/activate && python \
  /mnt/d/openclaw_bot/openclaw_bot/scripts/train_lora.py \
  --model Qwen/Qwen2.5-Coder-14B-Instruct-AWQ \
  --adapter-name openclaw-14b-v1 \
  --dataset /mnt/d/openclaw_bot/openclaw_bot/data/training/raw_dialogues.jsonl \
  --lora-rank 16 --epochs 3 \
  --batch-size 1 --grad-accum 16"
```

**Почему такие параметры для 14B:**
| Параметр | 7B значение | 14B значение | Причина |
|----------|------------|------------|---------|
| `--batch-size` | 4 (авто) | **1** | 14B занимает ~13GB VRAM, нет места для батча |
| `--grad-accum` | 4 (авто) | **16** | Компенсирует маленький батч — эффективный batcch = 16 |
| `--lora-rank` | 16 | 16 | Одинаково, rank не зависит от размера модели |
| `--epochs` | 3 | 3 | Можно снизить до 2 если датасет >800 примеров |

### Шаг 5.2 — Запуск обучения DeepSeek-R1

```bash

```

**Почему `--lora-rank 8` вместо 16?**  
DeepSeek-R1 уже хорошо умеет рассуждать — нам не нужно сильно менять его поведение, только направить на CS2/DMarket домен. Меньший rank = меньше вмешательство = сохраняем quality reasoning при меньшем риске забывания.

**Почему `--epochs 2`?**  
Research датасет (100–200 примеров) меньше основного. 2 эпохи достаточно; больше — риск переобучения на узких примерах.

## ❓ Частые вопросы

**Q: Бот замедлился — это нормально?**  
A: Да. Обучение занимает всю VRAM. Останови бот на время тренировки.

**Q: Обучение упало с OOM (out of memory)?**  
A: Уменьши `--batch-size` до 1 и/или `--max-seq-len` до 1024. Для 14B это критично — без `--batch-size 1` не запустится на 16GB.

**Q: ROUGE-1 = 0.10 — что делать?**  
A: Нужно больше данных (хотя бы 500 примеров) и проверить качество ручных примеров. Если примеры слишком однотипные — добавь разнообразия по категориям.

**Q: Как откатить адаптер если стало хуже?**  
A: `python await manager.swap_model("Qwen/Qwen2.5-Coder-14B-Instruct-AWQ")` — загрузит базовую модель без LoRA.

**Q: Можно ли обучать на диалогах на русском?**  
A: Да, Qwen2.5 отлично поддерживает русский язык. Более того — если бот общается с пользователями по-русски, именно русскоязычный датасет нужен для дообучения.

**Q: Сколько примеров нужно для каждой категории?**  
A: Минимум 20–30 примеров на категорию для базового обучения. Оптимально: 50–100 на категорию. Категории: анализ рынка, механики CS2, торговые стратегии, команды бота, инвестиции, технические вопросы.

**Q: Можно ли использовать ChatGPT/Claude для генерации обучающих примеров?**  
A: Да, но осторожно. Примеры от AI должны быть проверены вручную на фактическую точность — цены, даты, паттерны. Не вставляй сгенерированные данные о ценах без проверки через DMarket API. Лучший источник — реальные диалоги с ботом.

**Q: Как часто нужно переобучать модель?**  
A: Начальное обучение → когда датасет достигнет 200+ примеров. Потом переобучение каждые 2–4 недели при накоплении 100+ новых качественных примеров. После крупных изменений рынка (новая операция CS2, смена мета) — досрочное переобучение с новыми актуальными примерами.

**Q: Loss не убывает, застрял на 2.0 — что делать?**  
A: Возможные причины: 1) Некорректный формат датасета — проверь JSON. 2) Learning rate слишком высокий — добавь `--lr 1e-5`. 3) Все примеры слишком похожи — увеличь разнообразие. 4) Адаптер не применился — проверь что `peft` версии совместимы.

**Q: Можно ли обучать несколько адаптеров параллельно?**  
A: Нет. На 16GB GPU только одно обучение одновременно. Последовательно: сначала 7B v1, затем 7B v2 с улучшенным датасетом, затем 14B v1 и так далее.

**Q: Что делать с примерами где бот ошибся?**  
A: Это ценнейшие данные! Исправь ответ вручную и добавь исправленную пару в датасет. Это называется RLHF-lite (reinforcement from human feedback) — обучение на коррекции ошибок.

**Q: Как узнать какие слои модели обучаются через LoRA?**  
A: `bash wsl bash -c "source /mnt/d/vllm_env/bin/activate && python3 -c \"from peft import PeftConfig; c = PeftConfig.from_pretrained('/mnt/d/lora_adapters/openclaw-7b-v1'); print(c.target_modules)\""` — выведет список слоёв (обычно q_proj, v_proj, k_proj, o_proj).
